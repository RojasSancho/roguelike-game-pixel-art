# roguelike-game-pixel-art
My first personal game project: a roguelike inspired by '20 Minutes Till Dawn' with pixel-art aesthetics.

# Proyecto Personal: Roguelike Pixel Art

## Descripción
Este es mi primer proyecto de videojuego personal, inspirado en *20 Minutes Till Dawn*.  
El objetivo es crear un **roguelike interactivo** con estética **pixel art**, que sea divertido y visualmente atractivo.  
El proyecto servirá como **pieza de portfolio**, mostrando mis habilidades de programación, diseño y desarrollo de software.

### Objetivos del proyecto
- Crear un juego jugable con mecánicas básicas de roguelike: movimientos, enemigos, habilidades y oleadas.
- Mantener un estilo visual pixel art, inspirado en personajes coloridos y distintivos de anime (originales, no copiados de propiedad intelectual).
- Desarrollar el proyecto de manera incremental, agregando características y mejoras con el tiempo.
- Aprender y practicar buenas prácticas de programación orientada a objetos y estructuras de datos.
- Tener un proyecto que pueda crecer y escalar en complejidad, pero que se mantenga manejable como primer videojuego.

## Stack Tecnológico
- **Lenguaje de programación:** Python (POO, lógica de juego, estructuras de datos)
- **Motor / Librería:** [Pygame](https://www.pygame.org/news) o [Arcade](https://arcade.academy/) (para manejo de sprites, gráficos y eventos)
- **Editor / IDE:** VS Code (ya configurado con Git y GitHub)
- **Control de versiones:** Git + GitHub
- **Arte:** Pixel art hecho por mí, usando herramientas como:
  - [Aseprite](https://www.aseprite.org/) (opcional, de pago)  
  - [Piskel](https://www.piskelapp.com/) (gratuito, online)
- **Otros recursos:** Documentación de Python y librerías de juego, referencias de mecánicas roguelike, tutoriales de pixel art.

## Motivación
- Hacer un proyecto personal que sea **divertido y motivador**.
- Crear algo que pueda **mostrar en mi portfolio**.
- Aprender conceptos útiles de ingeniería de software y desarrollo de videojuegos, sin fricción excesiva por el lenguaje.
- Explorar pixel art y estética visual propia, de manera progresiva.
- Experimentar con interactividad y jugabilidad desde el inicio, antes de enfocarme en gráficos avanzados.

## Estado Actual
- Nivel en Python: Bajo-intermedio (POO)
- Nivel en C: Bajo-intermedio (structs, punteros, iniciando memoria dinámica)

## Próximos Pasos
1. Instalar Python 3.11+ y librería Pygame o Arcade.
2. Crear repositorio GitHub y configurar VS Code.
3. Comenzar con la estructura básica del juego:
   - Loop principal
   - Eventos de teclado
   - Movimiento de personaje
   - Enemigos simples
4. Crear primeros sprites pixel art simples (jugador y enemigos).
5. Iterar añadiendo mecánicas roguelike:
   - Oleadas de enemigos
   - Power-ups / habilidades
   - Puntaje / niveles
6. Pulir estética visual y animaciones simples.
7. Documentar y versionar cada avance.

## Notas importantes
- Todo arte usado será **original** para evitar problemas de copyright.
- El proyecto está pensado como **escalable**, con la posibilidad de añadir más mecánicas y refinamiento visual con el tiempo.
- Se prioriza la **interactividad y jugabilidad** por encima de gráficos complejos inicialmente.
- Este proyecto puede servir de **puente hacia futuros proyectos más avanzados**, incluso en C++ si se decide migrar o portar el juego más adelante.
